- [ ] **Optimize system prompts for Gemini (both dradamblack and drlipokatz).** A review against Gemini prompt engineering best practices found several issues in the system prompts. Apply all changes to both clients ŌĆö dradamblack (EN) and drlipokatz (HE). Use the `qabu-prompt-engineer-gemini` skill when making these changes.
    - **Add few-shot examples to both gatekeepers (widget + facebook_comments).** Gemini docs say "prompts without examples are likely less effective." Classification tasks benefit hugely from examples. Add 2-3 positive examples to each gatekeeper showing the expected inputŌåÆoutput. For the widget gatekeeper: one business question ŌåÆ ESCALATE, one greeting ŌåÆ short warm reply, one gibberish ŌåÆ IGNORE. For the facebook_comments gatekeeper: one supportive comment ŌåÆ warm reply, one spam ŌåÆ IGNORE, one question ŌåÆ ESCALATE.
    - **Add few-shot examples to facebook_comments main prompt.** Currently has zero examples. Add 1-2 examples showing the expected tone, length (2-3 sentences), and pattern of directing to private message or phone for medical details.
    - **Add explicit grounding instruction to widget.main.** Currently says "Never fabricate information that is not in the knowledge base" which is good but not strong enough for Gemini. Add the recommended grounding block near the top of the Critical Rules section: "Reply only using the information provided in the knowledge base. Do not use your own knowledge. If the answer is not explicitly in the knowledge base, state that the information is not available."

    Not urgent ŌĆö `.js` is a big improvement over `.json` already. (added 2026-03-17)
